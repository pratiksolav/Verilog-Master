Code Converter - Binary to XS3

The excess-3 (or XS3) code is used to express decimal numbers.

It is a self-complementary code and numerical system which has biased representation.

It is generally used for arithmetic operations to overcome problems faced while using BCD code.

Excess-3 arithmetic uses a different algorithm than a normal non-biased binary positional number system.

Excess-3 codes are unweighted and can be obtained by adding 3 to each decimal digit.

Further, it can be represented by using a 4-bit binary number for each digit.

For direct conversion of a binary number to its excess-3 code equivalent , we can add 0011 to the binary number.

The 1’s complement of an excess-3 code is the excess-3 code for the 9’s complement of the decimal number equivalent of the original number.

For e.g.

Take number = 2 , its excess-3 code is 0101 ;

1’s complement of 0101 is ‘1010’ (1111 - 0101) ;

‘1010’ is excess-3 code for decimal number = 7;

7 is the 9’s complement for 2 ( 9-2 = 7) ;

Hence , Code Converters can be useful in finding the Excess-3 Code Equivalent for the given Binary Number.

You can use Karnaugh Maps to solve for XS3 Code Outputs - E3 , E2, E1, E0.

Give it a try.

We will obtain Boolean Expression as :

E3 = B2.(B0 + B1) + B3 ;

E2 = (B2)'.(B0 + B1) + (B0)'.(B1)'.B2;

E1 = (B0 ⊕ B1)' ;

E0 = (B0)' ;

Verilog Codes:


Gate Level:

module binary_to_xs3_gate(bin , exc);
    input [3:0]bin;
    output [3:0]exc;
    wire [2:0]bin_not;
    wire w1 ,w2 , w3 , w4;
    
    not N0(bin_not[0] , bin[0]);
    not N1(bin_not[1] , bin[1]);
    not N2(bin_not[2] , bin[2]);
    
    or O1(w1 , bin[1] , bin[0] );
    and A1(w2 , bin[2] , w1);
    and A2(w3 , bin_not[2] , w1);
    and A3(w4 , bin[2] , bin_not[1] , bin_not[0]);
    
    or O2(exc[3] , w2 , bin[3]);
    or O3(exc[2] , w3 , w4);
    xnor X1(exc[1] , bin[1] , bin[0]);
    not N3(exc[0] , bin[0] );
    
    
endmodule

Dataflow Level:

module binary_to_xs3_dataflow( bin , exc );
    input [3:0]bin;
    output [3:0]exc;
        
    assign exc[3] = ( bin[2]&(bin[0] | bin[1]) ) | bin[3] ;
    assign exc[2] = ( (bin[0] | bin[1])&(~bin[2]) ) | ((~bin[0]) &(~bin[1]) & bin[2]) ;
    assign exc[1] = ~(bin[1]^bin[0]);
    assign exc[0] = ~bin[0];
        
endmodule

Behavioural Level:

module binary_to_xs3_behavioural( bin , exc );
    input [3:0]bin;
    output reg [3:0]exc;
            
    always@(bin)
        begin
            case(bin)
                4'b0000 : exc = 4'b0011;
                4'b0001 : exc = 4'b0100;
                4'b0010 : exc = 4'b0101;
                4'b0011 : exc = 4'b0110;
                4'b0100 : exc = 4'b0111;
                4'b0101 : exc = 4'b1000;
                4'b0110 : exc = 4'b1001;
                4'b0111 : exc = 4'b1010;
                4'b1000 : exc = 4'b1011;
                4'b1001 : exc = 4'b1100;
                4'b1010 : exc = 4'b1101;
                4'b1011 : exc = 4'b1110;
                4'b1100 : exc = 4'b1111;
                4'b1101 : exc = 4'bxxxx;
                4'b1110 : exc = 4'bxxxx;
                4'b1111 : exc = 4'bxxxx;
            endcase
        end
endmodule
